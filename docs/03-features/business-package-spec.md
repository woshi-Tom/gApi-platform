## 1. 套餐类型概述

### 1.1 免费用户 (Free)

| 项目 | 说明 |
|------|------|
| 获取方式 | 注册即得 |
| 有效期 | 7天 |
| 额度 | 50,000 tokens |
| API密钥 | 1个 |
| RPM限制 | 20次/分钟 |
| TPM限制 | 2K tokens/分钟 |
| 目的 | 引流试用，降低门槛 |

### 1.2 充值套餐 (Pay-as-you-go)

| 套餐 | 价格 | 有效期 | 额度 | RPM | TPM | 说明 |
|------|------|--------|------|-----|-----|------|
| 试用装 | ¥5.9 | 3天 | 100K | 30 | 3K | 首次试用体验 |
| 基础装 | ¥9.9 | 5天 | 300K | 60 | 6K | 临时项目需求 |
| 标准装 | ¥19.9 | 7天 | 1M | 100 | 10K | 一周稳定使用 |

**充值规则**：
- 多次购买按**购买时间顺序**消耗（先进先出 FIFO）
- 套餐消耗完任一条件（有效期到期 OR 额度用完）即为过期
- 新购买充值套餐：有效期重新计时，**额度叠加**（不清零）
- 只能消耗完当前套餐，才能继续消耗新套餐

**示例**：
```
用户购买套餐A (300K额度, 5天) → 购买套餐B (1M额度, 7天)
消耗顺序: A(先用) → 用完/过期 → B(后用)
```

**通知机制**：
- 套餐即将用完（剩余<10%）→ 邮件通知
- 套餐已用完 → 邮件通知 + 自动切换下一套餐

### 1.3 VIP会员 (VIP Subscription)

| VIP白银 | ¥59.9 | 1M | 200 | 100K | 3个 |
| VIP黄金 | ¥99.9 | 不限量 | 500 | 500K | 5个 |

**管理后台配置项**：
- 等级名称、图标

- 额度（支持设置具体数值或"不限量"）
- RPM/TPM限制
- API密钥数量
- 是否开启专属通道

## 3. 用户转化路径

```
┌─────────────────────────────────────────────────────────────┐
│                      用户生命周期                              │
└─────────────────────────────────────────────────────────────┘

[免费试用] 7天 → [临时充值] 3-7天 → [VIP订阅] 30天 → [长期会员]
    │              │                │
    └──────────────┴────────────────┘
              频繁充值用户 → 引导升级VIP
```

**转化策略**：
1. 免费7天吸引用户注册
2. 快到期时推送充值套餐（3-7天）
3. 频繁充值用户引导升级VIP
4. VIP到期前推送续费
5. VIP额度不足时，引导购买充值套餐补充

## 4. 额度消耗规则

### 4.1 消耗优先级

```
用户发起API请求
    ↓
┌─────────────────────────────────────┐
│ 检查免费额度 (free_quota)            │
│   → 优先消耗免费额度                │
│   → free_quota -= 本次消耗量        │
└─────────────────────────────────────┘
    ↓ (免费额度不足或过期)
┌─────────────────────────────────────┐
│ 检查充值套餐 (按购买顺序)             │
│   → 消耗当前有效套餐                │
│   → 当前套餐用完后自动切换下一套餐   │
└─────────────────────────────────────┘
    ↓ (所有充值套餐用完)
┌─────────────────────────────────────┐
│ 检查VIP额度 (vip_quota)             │
│   → 消耗VIP额度                    │
│   → vip_quota -= 本次消耗量         │
└─────────────────────────────────────┘
    ↓ (所有额度不足)
┌─────────────────────────────────────┐
│ 拒绝请求，返回配额不足错误           │
└─────────────────────────────────────┘
```

**消耗原则**：
- 先进先出（FIFO）：按购买时间顺序消耗充值套餐
- 不跳级：必须用完当前套餐才能用下一个
- VIP补充：VIP额度不足时可购买充值套餐

### 4.2 数据库字段设计

```sql
-- 用户表扩展字段
level                    VARCHAR(20)     -- free/vip_bronze/vip_silver/vip_gold
vip_expired_at          TIMESTAMP       -- VIP到期时间
vip_quota               BIGINT          -- VIP额度
vip_package_id          INT             -- VIP套餐ID

-- 充值套餐表新增字段
valid_days              INT             -- 有效期天数
expired_at              TIMESTAMP       -- 到期时间（购买时计算）

-- 用户充值记录表（用于FIFO顺序）
id                      BIGINT PRIMARY KEY
user_id                 INT
package_id              INT
quota                   BIGINT          -- 额度
remaining_quota         BIGINT          -- 剩余额度
order_no                VARCHAR(50)     -- 关联订单号
status                  VARCHAR(20)     -- active/used/expired
created_at              TIMESTAMP
expired_at              TIMESTAMP       -- 到期时间
```

### 4.3 套餐切换逻辑

```
1. 查询用户所有充值套餐（按创建时间排序）
2. 找到 status='active' 且 expired_at > now() 的套餐
3. 消耗当前套餐额度
4. 如果当前套餐用完（remaining_quota=0 或 expired_at < now()）
   → 更新status='used' 或 'expired'
   → 发送邮件通知
   → 自动切换到下一个套餐
5. 所有套餐用完后，切换到VIP额度或免费额度
```

## 6. VIP续费规则

### 6.1 VIP有效期判定

**VIP结束条件（满足任一即结束）**：

| 条件 | 说明 |
|------|------|
| 时间到期 | vip_expired_at < 当前时间 |
| 额度用完 | vip_quota <= 0 |

**示例**：
```
用户开通VIP 30天/1M额度
第20天用完1M额度 → VIP立即结束
剩余10天不继承，额度清零
```

### 6.2 续费时机

- VIP到期前7天：前端提示续费
- VIP到期当天：强制续费提醒
- VIP额度<20%时：提示续费或购买充值套餐
- VIP结束后：VIP额度清零，保留充值/免费额度

### 6.3 续费行为

| 场景 | 处理方式 |
|------|----------|
| VIP有效期内续费 | 有效期 = 当前到期时间 + 30天，VIP额度 = 当前剩余额度 + 套餐额度 |
| VIP已结束后续费 | 重新开通VIP，有效期从续费当天开始计算30天，额度 = 套餐额度 |

**注意**：VIP到期时间按用户开通日期计算，不绑定每月1日

## 7. 充值套餐有效期处理

### 7.1 到期处理

- 充值额度到期后状态变为 expired
- 不影响VIP额度（如有）
- 发送邮件通知用户
- 自动切换到下一个有效套餐

### 7.2 叠加规则

| 场景 | 行为 |
|------|------|
| 购买新充值套餐 | 有效期重新计时，**额度叠加** |
| 已有充值额度+购买 | 有效期重新计时，额度增加 |
| 多个充值套餐 | 按购买顺序消耗（先进先出） |
| VIP+充值同时存在 | 独立计时，独立消耗 |

### 7.3 VIP用户购买充值

**允许**：VIP用户可以购买充值套餐作为额度补充

**VIP专属优惠**：VIP用户购买充值套餐享受9折优惠

**生效时机**：VIP额度用完后自动生效

**管理后台配置**：
- VIP用户充值折扣比例（默认90%，可调整）
- 优惠说明文案
