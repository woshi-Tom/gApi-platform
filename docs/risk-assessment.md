## 🔴 高风险（立即行动）

### R1: OpenAI API 封禁中国

| 项目 | 内容 |
|------|------|
| **风险描述** | OpenAI 可能封禁中国地区 IP 或 API 访问，导致渠道不可用 |
| **发生概率** | 高（已有先例） |
| **影响程度** | 高（影响核心功能） |
| **风险等级** | 🔴 **高** |
| **缓解策略** | 1. 优先集成 Azure OpenAI（合规渠道）<br>2. 增加国产模型支持（智谱、通义千问）<br>3. 支持 NVIDIA NIM 免费模型<br>4. 支持通过代理转发 |
| **应急预案** | 立即切换到 Azure OpenAI 或国产模型，公告通知用户 |
| **责任人** | 后端工程师 |
| **截止日期** | Day 5（渠道管理完成时） |

### R2: 配额扣减竞态条件

| 项目 | 内容 |
|------|------|
| **风险描述** | 高并发下配额扣减可能不一致，导致用户超支或扣款错误 |
| **发生概率** | 中（高并发场景下） |
| **影响程度** | 高（财务损失） |
| **风险等级** | 🔴 **高** |
| **缓解策略** | 1. 使用数据库行级锁（SELECT ... FOR UPDATE）<br>2. 预扣机制（先扣后调）<br>3. 事后调整逻辑<br>4. 限制单用户并发数 |
| **应急预案** | 回滚最近 1 小时的扣款记录，人工审计 |
| **责任人** | 后端工程师 |
| **截止日期** | Day 7（API 代理完成时） |

### R3: 项目延期超过 50%

| 项目 | 内容 |
|------|------|
| **风险描述** | 实际开发时间超过预期 50%（18 天 vs 12 天） |
| **发生概率** | 中（技术复杂度高） |
| **影响程度** | 高（影响整体进度） |
| **风险等级** | 🔴 **高** |
| **缓解策略** | 1. 优先完成 P0 任务<br>2. P3 任务延后迭代<br>3. 减少非核心功能<br>4. 每日站会追踪 |
| **应急预案** | 缩减范围，先上线 MVP 版本 |
| **责任人** | 项目经理 |
| **截止日期** | 持续监控 |

### R4: 技术栈不熟悉

| 项目 | 内容 |
|------|------|
| **风险描述** | 开发者对 Flask + SQLAlchemy + 多适配器模式不熟悉 |
