## 📝 开发规范

### 功能开发流程

```
1. 理解需求 → 创建Todo
2. 实现代码 → 构建测试
3. 功能体验 → 发现bug
4. 修复bug → 重新测试
5. 验证通过 → Git提交
6. 提交完成 → 移除Todo
```

### 提交规范

```bash
# 功能完成
git commit -m "feat: 实现xxx功能"

# Bug修复
git commit -m "fix: 修复xxx问题"

# 文档更新
git commit -m "docs: 更新xxx文档"
```

### 构建检查

每次实现后必须验证：

```bash
# 后端
cd backend && go build ./...

# 前端
cd frontend && npm run build
```
